# AI重设计提示词文档

## 项目概述

**项目名称**: 爬山助手 (Hiking Assistant)
**平台**: Flutter Mobile/Web App
**设计风格**: 有机生物亲和设计 (Organic Biophilic Design)
**核心品牌色**: 森林绿 (Forest Green)

---

## 设计系统基础

在重设计每个页面前，请遵循以下设计规范：

### 色彩体系
- **主色**: `#1B4D3E` (Forest - 深森林绿), `#2E7D62` (ForestLight - 浅森林绿)
- **辅助色**: `#F59E0B` (Sun - 暖黄), `#EF4444` (Error - 错误红), `#8B5CF6` (Lavender - 薰衣草紫)
- **背景色**: `#FAFAF9` (Paper - 米白纸张质感), `#F5F5F4` (PaperDark - 浅灰)
- **文字色**: `#1C1917` (Ink - 深灰), `#78716C` (InkMuted - 中灰), `#A8A29E` (InkLight - 浅灰)

### 间距规范
- `sm`: 8px, `md`: 16px, `lg`: 24px, `xl`: 32px
- 圆角: `radiusMd`: 12px, `radiusLg`: 16px, `radiusXl`: 24px, `radiusFull`: 999px

### 字体层级
- `display`: 48px+ (超大标题)
- `headline`: 24-32px (页面标题)
- `title`: 18-20px (卡片标题)
- `body`: 14-16px (正文)
- `label`: 12-14px (标签/辅助文字)
- `data`: 12px (数据展示)

### 设计风格特点
1. **卡片式布局**: 白色背景 + 圆角 + 软阴影
2. **渐变头部**: 森林绿渐变作为品牌识别
3. **毛玻璃效果**: BackdropFilter blur + 半透明背景
4. **自然纹理**: 等高线、山峦轮廓、地形元素
5. **一致的状态处理**: loading/data/error 三态展示

### 页面 1: 登录页 (LoginScreen)

**文件路径**: `lib/features/auth/presentation/screens/login_screen.dart`

#### 当前功能
- 用户认证入口页面
- Google OAuth 登录按钮
- 游客模式登录入口
- 自动检查认证状态并跳转

#### 当前UI结构
- Logo容器 (72x72圆角方形，forest色半透明)
- 标题「爬山助手」+ 副标题「你的智能户外伙伴」
- 主按钮「开始使用」
- 副按钮「游客模式」
- 底部协议提示

#### 重设计提示词

```
请重新设计登录页面，要求：

**核心目标**: 打造一个令人印象深刻的品牌入口，传达户外探险的品牌精神。

**布局结构**:
- 全屏沉浸式设计，利用自然风景图片或渐变作为背景
- Logo和标题居中偏上，但不要过于对称（避免模板感）
- 按钮组底部堆叠，主按钮视觉突出

**视觉风格**:
- 背景使用深色森林绿渐变 + 微妙的纹理（等高线/地形图案）
- Logo使用山峦图标 + 动态效果（如呼吸灯、微动画）
- 按钮使用毛玻璃效果，主按钮带品牌色渐变
- 添加底部山峦剪影装饰

**交互增强**:
- Logo区域添加微妙的scale动画
- 按钮hover时改变透明度和边框样式
- 游客模式按钮更低调（透明度降低，边框细线）
- 考虑添加onboarding滑动预览（可选）

**避免**:
- 不要使用纯白背景 + 单一品牌色按钮的模板设计
- 不要使用过多的圆角（保持锐利和现代感）
- 不要让Logo区域过于小气，要大气

**参考风格**: National Geographic、AllTrails、户外探险类App的登录页
```

### 页面 2: 首页 (HomeScreen)

**文件路径**: `lib/features/hiking/presentation/screens/home_screen.dart`

#### 当前功能
- 应用主入口，展示天气卡片、路线推荐、快捷操作、最近活动

#### 当前UI结构
- 天气Hero卡片 (160px高度渐变)
- 搜索栏 + 筛选按钮
- 快捷操作胶囊按钮（找路线/导航/记录/识植物）
- 推荐路线卡片列表
- 最近活动列表

#### 重设计提示词

```
请重新设计首页，要求：

**核心目标**: 信息密度适中，层次分明，一眼看到关键信息（天气+推荐路线）。

**布局结构**:
- 顶部天气区域：考虑使用不规则形状（非矩形卡片），如带弧形底部
- 快捷操作：使用Bento网格布局而非横向胶囊
- 路线推荐：卡片网格（2列）或单列大卡片，不要过于紧凑
- 滚动体验：天气区域可折叠或视差滚动

**视觉风格**:
- 天气卡片：保持渐变但增加层次（云层纹理、天气图标动态）
- 快捷操作：使用毛玻璃小卡片，图标+短文字，每个卡片颜色略有区分
- 路线卡片：保持现有设计但增强图片质量感，底部信息行使用更小的标签
- 空状态：使用插画而非纯图标

**交互增强**:
- 天气卡片点击展开更多天气详情
- 快捷操作卡片添加hover/scale微动画
- 路线卡片点击时平滑过渡到详情页
- 下拉刷新使用品牌色进度指示器

**避免**:
- 不要让快捷操作变成一排相同胶囊的模板设计
- 不要使用过于统一的卡片尺寸（应有大小对比）
- 不要在天气卡片上堆砌太多数据

**参考风格**: Airbnb首页、Strava活动页、天气类App的Hero区
```

### 页面 3: 地图页 (MapScreen)

**文件路径**: `lib/features/hiking/presentation/screens/map_screen.dart`

#### 当前功能
- OpenStreetMap底图、用户位置、路线起点标记
- 位置搜索、地图样式切换、轨迹记录控制
- 底部可拖拽路线列表

#### 当前UI结构
- FlutterMap + TileLayer + MarkerLayer

- 搜索栏（毛玻璃）+ 图层按钮
- DraggableScrollableSheet 路线列表
- FAB组（开始/暂停/停止轨迹记录）

#### 重设计提示词

```
请重新设计地图页，要求：

**核心目标**: 地图为主，控件为辅，保持专业户外地图App的感觉。

**布局结构**:
- 地图占据全屏，控件通过浮层叠加
- 搜索栏精简：只保留搜索框和定位按钮，去除过多装饰
- 底部路线列表：使用抽屉式设计，默认最小化，只有一条提示线
- FAB组：合并为单个复合按钮（主按钮 + 展开子按钮）

**视觉风格**:
- 地图控件使用深色毛玻璃（更适合地图背景）
- 路线标记：使用更精致的MountainMarker，带阴影和标签
- 用户位置：使用脉冲动画效果的定位圈
- 轨迹线：使用渐变颜色（根据海拔变化）

**交互增强**:
- 长按地图添加自定义标记
- 路线标记点击弹出预览卡片（不跳转）
- 轨迹记录开始时显示实时数据浮窗
- 支持地图手势（双击放大、双指缩放）

**避免**:
- 不要让顶部控件占据太多空间
- 不要使用过多不同颜色的FAB
- 不要在地图上堆砌太多标记（要有聚合策略）

**参考风格**: Komoot、MapOut、Strava地图页
```

## 使用指南

### 如何使用这些提示词

1. **选择页面**: 定要重设计的页面
2. **复制提示词**: 复制对应页面的提示词内容
3. **调整细节**: 根据实际需求调整（如添加特定功能、调整视觉风格）
4. **提供给AI**: 将提示词提供给设计AI（如Claude、Midjourney、或UI设计工具）
5. **迭代优化**: 根据AI输出结果，调整提示词并重新生成

### 提示词增强技巧

- 添加具体参考图片/网站链接
- 指定设计工具（如"请生成Figma组件代码"）
- 添加技术约束（如"使用Flutter Material 3"）
- 添加品牌约束（如"保持森林绿品牌色")

### 品牌一致性检查

重设计后请检查：
- [ ] 使用了森林绿品牌色系
- [ ] 保持了纸质质感背景
- [ ] 使用了统一间距规范
- [ ] 卡片使用了软阴影效果
- [ ] 渐变头部风格一致
- [ ] 避免了模板感设计
